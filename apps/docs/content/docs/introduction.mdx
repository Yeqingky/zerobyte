Zerobyte 是一款备份自动化工具，帮助你跨多个存储后端保护数据。它基于 [Restic](https://restic.net/) 构建，提供了一个现代化的 Web 界面来调度、管理和监控加密备份。

<Callout type="warn">
  Zerobyte 仍处于 0.x.x 版本，各版本之间可能会有重大变化。核心功能正在积极开发中
</Callout>

## 支持的存储

### 卷来源（要备份的内容）

卷代表您想要保护的数据：

- **本地目录**，挂载到容器中的目录
- **NFS**，网络文件系统共享（v3、v4、v4.1）
- **SMB/CIFS**，Windows 和 Samba 共享
- **WebDAV**，兼容 WebDAV 的存储（Nextcloud、ownCloud 等）
- **SFTP**，SSH 文件传输协议服务器
- **Rclone**，通过 rclone 支持 40 多种云存储提供商（Google Drive、Dropbox、OneDrive 等）

### 仓库目标（备份存储位置）

仓库是存储备份快照的加密位置：

- **本地目录**，将备份存储在本地磁盘
- **兼容 S3 的存储**，Amazon S3、MinIO、Wasabi、Backblaze B2、DigitalOcean Spaces 等
- **Cloudflare R2**，兼容 S3，零出站费用
- **Google Cloud Storage**，谷歌云存储服务
- **Azure Blob Storage**，微软 Azure 存储
- **SFTP**，可通过 SSH 访问的远程服务器
- **REST 服务器**，Restic REST 服务器协议
- **Rclone 远程存储**，通过 rclone 支持 40 多种云存储提供商

## 工作原理

Zerobyte 基于三个核心概念运行：

<Steps>
  <Step>

### 卷

定义**卷**作为要备份的源数据。这些可以是本地目录、网络共享或通过各种协议挂载的云存储。Zerobyte 会自动处理挂载、健康监控和自动恢复。

  </Step>

  <Step>

### 仓库

创建**仓库**作为加密的存储目标。仓库使用 Restic 的去重和加密技术高效安全地存储您的备份。可以选择本地磁盘、云对象存储或 40 多种 rclone 后端。

  </Step>

  <Step>

### 备份任务

配置**备份任务**，将卷与仓库连接起来，并设置调度规则和保留策略。任务将根据您的计划自动运行，并在 Web 界面中实时监控进度。

  </Step>
</Steps>

<Callout type="info">
  所有备份均使用 Restic 的加密技术进行加密。您的加密密码将被安全存储，绝不会离开您的服务器。
</Callout>

## 使用场景

<Accordions>
  <Accordion title="家庭服务器备份">
    将家庭服务器数据（照片、文档、媒体）自动备份到云存储或外部硬盘，并设置调度和保留策略。

  <Accordion title="多站点备份">
    集中管理多个位置的备份，将不同来源的数据备份到不同类型的仓库。
  </Accordion>
</Accordions>

## 为什么选择 Restic？

Zerobyte 基于 [Restic](https://restic.net/) 构建，因为它提供了：

- **快速高效**，基于内容定义分块的增量备份
- **设计安全**，强加密和经过认证的快照
- **去重**，仅存储唯一数据，节省空间
- **校验**，内置完整性检查，让您安心
- **跨平台**，支持不同操作系统
- **维护良好**，活跃的开源项目，定期更新
