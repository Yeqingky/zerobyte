Zerobyte 是基于组织作用域的。仓库、卷、备份计划、通知目标、SSO 提供商和邀请都属于一个组织。

## 组织的控制范围

- 每个请求都在一个活跃的组织上下文中运行。
- 资源列表和变更操作会被过滤到该活跃组织。
- 成员身份决定了谁可以访问组织范围的特性。
- **设置 > 组织**中显示的组织 ID 与自动配置使用的 ID 相同。

## 组织如何创建

对于普通的本地注册，如果新用户没有现有成员身份，Zerobyte 会创建一个名为类似 `Alice 的工作区` 的默认工作区，并将该用户设为 `owner`。

被邀请的 SSO 用户行为不同：Zerobyte 在首次成功 SSO 回调时会在被邀请的组织中创建其成员身份，因此他们会直接进入该组织，而不是获得一个单独的默认工作区。

## 组织角色

| 角色 | 实际含义 |
| --- | --- |
| `member` | 标准组织成员。可以使用组织，但无法访问组织管理操作，如成员管理或 SSO 设置。 |
| `admin` | 可以访问**组织**设置标签页、管理成员、管理邀请、切换 SSO 自动关联，以及下载活跃组织的恢复密钥。 |
| `owner` | 拥有所有管理员能力，此外是注册新 SSO 提供商的必需角色。Owner 不能通过当前的组成员管理端点被降级或移除。 |

<Callout type="info">
  Zerobyte 还有一个实例级的用户角色 `admin`，用于全局任务，如注册控制和用户管理。该全局角色与 `member`、`admin` 和 `owner` 等组织角色是分开的。
</Callout>

## 邀请

组织邀请包含四个重要的数据项：

- 被邀请者的电子邮件地址
- 接受后授予的角色
- 邀请状态
- 过期时间

在当前 UI 中，邀请管理位于**设置 > 组织 > 单点登录**内，因为 SSO 访问是基于邀请的。

### 通过邀请分配角色

新的邀请可以立即分配以下组织角色之一：

- `member`
- `admin`
- `owner`

这一点很重要，因为当前的**成员**表格只允许你在 `member` 和 `admin` 之间切换现有成员。如果你想添加另一个 owner，请从邀请流程中邀请该用户为 `owner`。

## 成员管理行为

- 组织管理员和 owner 可以将 `member` 提升为 `admin`。
- 组织管理员和 owner 可以将 `admin` 降级为 `member`。
- Owner 不能通过成员管理 API 被降级。
- Owner 不能通过成员管理 API 被移除。
- 移除成员会立即撤销他们对组织资源的访问权限。

如果被移除的用户属于另一个组织，Zerobyte 会将其活跃会话重新归属到备用组织成员身份。如果他们不属于任何其他组织，Zerobyte 会撤销其会话，他们在被重新添加后必须重新登录。

## 组织 ID

**设置 > 组织**中显示的**组织 ID**主要是一个参考值，但对于运维管理的设置很重要，因为自动配置条目必须指向一个现有的组织 ID。

如果你以声明式方式管理仓库和卷，请参阅[自动配置](/docs/guides/provisioning)。
