<Callout type="info">
  本指南假设您已经[安装了 Zerobyte](/docs/installation) 并且可以访问 Web 界面。
</Callout>

## 概述

设置备份包含三个关键步骤：

<Steps>
  <Step>

### 添加卷

定义要备份的源数据

</Step>

  <Step>

### 创建仓库

为备份设置加密的存储目标

</Step>

  <Step>

### 配置备份任务

## 首次设置

在全新的 Zerobyte 安装中（尚无任何用户），应用会从引导流程开始：

<Steps>
  <Step>

### 打开 Zerobyte

导航到您在安装时配置的 `BASE_URL` 中指定的 URL。通常是 `http://<服务器IP>:4096`。

</Step>

  <Step>

### 创建管理员用户

在**引导**页面上，输入：

- **电子邮箱**
- **用户名**
- **密码**
- **确认密码**

然后点击**创建管理员用户**。

</Step>

  <Step>

### 下载恢复密钥

管理员用户创建后，Zerobyte 会重定向到**下载恢复密钥**页面。

在**确认您的密码**中输入您的账户密码，然后点击**下载恢复密钥**。

</Step>

  <Step>

### 继续前往卷

恢复密钥下载完成后，Zerobyte 会重定向到**卷**页面。本指南的后续部分将从此处开始。

</Step>
</Steps>

<Callout type="warn">
  **重要**：请安全保存下载的恢复密钥文件。它包含恢复备份所需的 Restic 密码。如果您同时丢失了服务器访问权限和此文件，您的备份将无法恢复。有关完整模型（包括使用自定义密码的导入仓库），请参阅[恢复密钥和仓库密码](/docs/guides/recovery-key-and-repository-passwords)。
</Callout>

<Callout type="info">
  引导流程完成后，后续访问将使用普通的**登录**页面，而不是引导流程。
</Callout>

## 步骤 1：添加卷

**卷**代表您要备份的源数据。Zerobyte 支持多种卷类型，包括本地目录、NFS 共享、SMB 共享、WebDAV、SFTP 和 rclone 挂载。

### 添加本地目录

在本快速入门中，我们将添加一个本地目录。首先，确保该目录已挂载到您的 Zerobyte 容器中。

#### 挂载目录

更新您的 `docker-compose.yml` 以挂载要备份的目录：

```yaml
services:
  zerobyte:
    image: ghcr.io/nicotsx/zerobyte:v0.37
    container_name: zerobyte
    restart: unless-stopped
    # ... 其他配置 ...
    volumes:
      - /etc/localtime:/etc/localtime:ro
      - /var/lib/zerobyte:/var/lib/zerobyte
      - /path/to/your/data:/data  # 添加此行
```

重启容器以应用更改：

```bash
docker compose down
docker compose up -d
```

#### 在 Zerobyte 中创建卷

<Steps>
  <Step>

### 导航到卷

登录后，您已经位于**卷**页面。如果您导航到了其他地方，请点击侧边栏中的**卷**。

</Step>

  <Step>

### 创建新卷

点击**创建卷**按钮

</Step>

  <Step>

### 配置卷

填写卷的详细信息：

- **名称**：给卷起一个描述性名称（例如："我的文档"）
- **后端**：保持设置为 `Directory`
- **目录路径**：使用文件系统浏览器选择已挂载的目录（例如：`/data`）

</Step>

  <Step>

### 保存卷

点击**创建**以保存卷配置

</Step>
</Steps>

<Callout type="info">
  目录浏览器显示的是 Zerobyte 服务器或容器文件系统中的路径，而不是直接显示宿主机文件系统。在上述 Docker 示例中，宿主机路径 `/path/to/your/data` 在 Zerobyte 中显示为 `/data`，因此这就是您在浏览器中选择的路径。
</Callout>

### 添加远程卷

Zerobyte 还支持远程卷类型：

<Tabs items={['NFS', 'SMB/CIFS', 'WebDAV', 'SFTP']}>
  <Tab value="NFS">
    配置 NFS 共享：

    - **服务器**：NFS 服务器主机名或 IP
    - **导出路径**：NFS 导出路径（例如：`/mnt/data`）
    - **端口**：NFS 端口（默认：`2049`）
    - **版本**：`3`、`4` 或 `4.1`
    - **只读模式**：可选，建议用于备份源
  </Tab>

  <Tab value="SMB/CIFS">
    配置 SMB/CIFS 共享：

    - **服务器**：SMB 服务器主机名或 IP
    - **共享**：共享名称
    - **访客模式**：可选，用于无需认证的共享
    - **用户名**：SMB 用户名
    - **密码**：SMB 密码
    - **SMB 版本**：`1.0`、`2.0`、`2.1` 或 `3.0`
    - **域**：Windows 域或工作组（可选）
    - **端口**：SMB 端口（默认：`445`）
  </Tab>

  <Tab value="WebDAV">
    配置 WebDAV 服务器：

    - **服务器**：WebDAV 服务器主机名或 IP
    - **路径**：服务器上的远程路径
    - **用户名**：WebDAV 用户名（可选）
    - **密码**：WebDAV 密码（可选）
    - **端口**：服务器端口（默认 `80`，HTTPS 为 `443`）
    - **使用 SSL/HTTPS**：启用安全的 HTTPS 连接
    - **只读模式**：可选，建议用于备份源
  </Tab>

  <Tab value="SFTP">
    配置 SFTP 服务器：

    - **主机**：SFTP 服务器主机名或 IP
    - **端口**：SFTP 端口（默认：22）
    - **用户名**：SSH 用户名
    - **密码**：SSH 密码（如果使用私钥则可选）
    - **私钥**：SSH 私钥（如果使用密码则可选）
    - **路径**：远程目录路径
    - **已知主机**：必需，除非您启用**跳过主机密钥验证**
  </Tab>
</Tabs>

<Callout type="info">
  远程卷需要完整的安装配置，包括 `SYS_ADMIN` 能力和 `/dev/fuse` 设备。详情请参阅[安装指南](/docs/installation)。
</Callout>

## 步骤 2：创建仓库

**仓库**是存储备份的加密目标。Zerobyte 支持多种仓库后端。

### 创建本地仓库

在本快速入门中，我们将创建一个本地仓库：

<Steps>
  <Step>

### 导航到仓库

点击侧边栏中的**仓库**

</Step>

  <Step>

### 创建新仓库

在新安装中，从空状态点击**创建仓库**

</Step>

  <Step>

### 配置仓库

填写仓库的详细信息：

- **名称**：描述性名称（例如："本地备份仓库"）
- **后端**：选择 `Local`
- **压缩模式**：`Auto (fast)` 是默认值，也是一个不错的起点
- **导入现有仓库**：如果是全新仓库，请保持未选中状态

</Step>

  <Step>

### 查看仓库目录

对于本地仓库，Zerobyte 将数据存储在仓库基本目录下，默认为 `/var/lib/zerobyte/repositories/{unique-id}`。只有在您想选择不同的基本目录时才点击**更改**。

</Step>

  <Step>

### 创建仓库

点击**创建仓库**。Zerobyte 会创建并初始化仓库，然后将您重定向到仓库详情页面。

</Step>
</Steps>

<Callout type="info">
  如果您选择了自定义的本地仓库目录，请确保它已从宿主机挂载到容器中。否则，容器重启时仓库数据可能会丢失。
</Callout>

<Callout type="info">
  如果您要导入现有仓库而不是创建新仓库，请在选择使用现有恢复密钥还是自定义仓库密码之前，阅读[恢复密钥和仓库密码](/docs/guides/recovery-key-and-repository-passwords)。
</Callout>

### 云存储仓库

在生产环境中，建议使用云存储进行异地备份：

<Tabs items={['兼容 S3', 'Google Cloud Storage', 'Azure Blob', 'rclone']}>
  <Tab value="S3-Compatible">
    配置兼容 S3 的存储（AWS S3、MinIO、Wasabi、Backblaze B2 等）：

    ```yaml
    Type: S3
    Endpoint: s3.amazonaws.com (or custom endpoint)
    Bucket: my-backup-bucket
    Access Key ID: AKIAIOSFODNN7EXAMPLE
    Secret Access Key: my-secret-access-key
    ```
  </Tab>

  <Tab value="Google Cloud Storage">
    配置 Google Cloud Storage：

    ```yaml
    Type: Google Cloud Storage
    Bucket: my-backup-bucket
    Project ID: my-project-id
    Service Account JSON: Paste the service account JSON
    ```
  </Tab>

  <Tab value="Azure Blob">
    配置 Azure Blob Storage：

    ```yaml
    Type: Azure Blob Storage
    Container: backups
    Account Name: myaccountname
    Account Key: my-account-key
    Endpoint Suffix: core.windows.net (optional)
    ```
  </Tab>

  <Tab value="rclone">
    配置基于 rclone 的存储（需要挂载 rclone 配置）：

    ```yaml
    Type: rclone
    Remote: gdrive (must exist in rclone config)
    Path: backups/zerobyte
    ```

    <Callout type="info">
      必须在宿主机上配置 rclone 并将其挂载到容器中。请参阅[安装 - 挂载 rclone 配置](/docs/installation#mounting-rclone-configuration)。
    </Callout>
  </Tab>
</Tabs>

<Callout type="info">
  如果您希望从环境变量或 Docker secrets 中获取凭据，请使用[预配置指南](/docs/guides/provisioning)。标准的卷和仓库表单目前期望直接输入实际密钥值并将其加密存储。
</Callout>

## 步骤 3：创建备份任务

**备份任务**将卷连接到仓库，并定义备份的运行时间和方式。

<Steps>
  <Step>

### 导航到备份

点击侧边栏中的**备份**

</Step>

  <Step>

### 开始新建备份任务

点击**创建备份任务**

</Step>

  <Step>

### 选择卷

选择您刚刚创建的卷。Zerobyte 随后会打开该卷的备份配置表单。

</Step>

  <Step>

### 配置基本设置

- **备份名称**：描述性名称（例如："每日文档备份"）
- **备份仓库**：选择您之前创建的仓库
- **备份频率**：选择任务应运行的频率

</Step>

  <Step>

### 配置调度

选择符合您备份需求的调度方式：

- **仅手动**：创建任务时不设置调度，按需手动运行
- **每小时**
- **每日**
- **每周**
- **特定日期**：选择一个或多个每月的日期
- **自定义（Cron）**：输入 cron 表达式

如果您选择了**每日**、**每周**或**特定日期**，请设置**执行时间**。

如果您选择了**每周**，还需选择星期几。

<Accordions type="single">
  <Accordion title="常见 Cron 模式">
    - `0 2 * * *` - 每天凌晨 2:00
    - `0 2 * * 0` - 每周日凌晨 2:00
    - `0 2 1 * *` - 每月 1 日凌晨 2:00
    - `0 */6 * * *` - 每 6 小时
    - `0 0 * * 1-5` - 工作日午夜
  </Accordion>
</Accordions>

</Step>

  <Step>

### 配置保留策略

定义备份的保留时长：

```yaml
Keep Daily: 7
Keep Weekly: 4
Keep Monthly: 6
Keep Yearly: 1
```

<Callout type="info">
  保留策略会自动清理旧备份以节省存储空间。Restic 的去重机制确保您只存储唯一数据。
</Callout>

</Step>

  <Step>

### 配置路径（可选）

默认情况下，会备份整个卷。您可以选择：

- 使用**备份路径**仅选择卷中的特定文件或文件夹
- 添加**额外的包含模式**，用于基于 glob 的包含规则
- 添加**排除模式**，跳过特定文件或文件夹（例如：`*.tmp`、`node_modules/**`）

</Step>

  <Step>

### 高级选项（可选）

如果您需要额外的 Restic 选项，请打开**高级**。

常见选项包括：

- **最大重试次数**：自动重试失败的备份运行
- **重试延迟**：重试失败运行前的等待时间
- **自定义 restic 参数**：每行传递一个受支持的 Restic 参数

</Step>

  <Step>

### 保存备份任务

点击**创建**以保存备份任务。Zerobyte 会直接将您重定向到备份任务详情页面。

</Step>
</Steps>

## 步骤 4：运行您的第一次备份

无需等待计划的执行时间，让我们立即运行备份：

<Steps>
  <Step>

### 打开备份任务详情

创建任务后，Zerobyte 会自动打开备份任务详情页面

</Step>

  <Step>

### 手动运行备份

点击**立即备份**

</Step>

  <Step>

### 监控进度

Zerobyte 会立即启动任务并在详情页面上更新状态。运行开始时会出现一个成功提示。

</Step>

  <Step>

### 等待完成

等待备份完成。当备份成功完成时，任务状态将变为**成功**，并且**快照**部分会出现一个新条目。

</Step>
</Steps>

<Callout type="info">
  首次备份时，所有数据都会被上传。由于 Restic 的增量备份和去重机制，后续备份会快得多。
</Callout>

## 步骤 5：验证您的备份

备份完成后，验证其是否成功：

<Steps>
  <Step>

### 打开快照

在**快照**部分，点击您刚刚创建的快照

</Step>

  <Step>

### 查看快照

您应该可以看到快照的详细信息，例如：

- 快照时间戳
- 总大小
- 新增数据 vs. 已存储数据
- 已处理文件数
- 耗时

</Step>

  <Step>

### 浏览备份内容

使用内嵌的文件浏览器检查快照内容。您可以：

- 浏览目录结构
- 查看文件大小
- 确认预期的文件是否存在

</Step>

  <Step>

### 测试恢复（可选）

为了安心，测试恢复一个文件：

- 从快照详情中点击**恢复**
- 选择**原始位置**或**自定义位置**
- 使用**全部恢复**或选择特定文件仅恢复所选内容
- 验证恢复的文件是否存在于目标位置

</Step>
</Steps>

<Callout type="warn">
  默认情况下，恢复的文件会被放回其原始位置。注意不要覆盖当前文件。如果需要，您可以指定一个替代的恢复路径。
</Callout>

## 了解备份状态

界面展示了两种相关的状态视图：

- 在**备份**列表中，状态点显示**活跃**、**已暂停**、**错误**、**警告**或**备份进行中**
- 在备份任务详情页面中，上次运行状态显示**✓ 成功**、**✗ 错误**、**! 警告**或**⟳ 进行中...**
- 如果备份任务尚未完成任何一次运行，详情页面将不显示上次运行值

<Callout type="info">
  点击任意备份任务可查看其快照历史、恢复操作、调度摘要和仓库链接。如果运行失败且界面未显示足够详细的信息，请使用 `docker compose logs -f zerobyte` 检查容器日志。
</Callout>

## 监控备份

Zerobyte 提供了多种监控备份健康状态的方式：

### 备份任务列表

**备份**页面显示：

- 备份任务名称
- 源卷和目标仓库
- 调度
- 上次备份时间
- 下次备份时间

### 备份任务详情

每个备份任务显示：

- 上次运行时间和状态
- 下次备份时间
- 快照历史
- 所选快照的快照统计信息
- 所选快照的恢复操作

### 仓库健康状态

仓库详情显示：

- 当前仓库状态（`healthy`、`error`、`unknown`、`doctor` 或 `cancelled`）
- 上次检查时间
- 压缩统计信息和快照数量
- Doctor 报告输出

<Callout type="info">
  当您需要进行仓库维护时，请在仓库详情页面中使用**运行 doctor**。当您只需要更新存储大小和快照指标时，请使用**压缩统计信息**中的刷新按钮。有关状态值和操作的确切含义，请参阅[仓库维护与状态](/docs/guides/repository-maintenance)。
</Callout>

## 常见操作

### 恢复数据

要从备份中恢复数据：

<Steps>
  <Step>

### 导航到备份

进入**备份**，选择包含所需数据的备份任务

</Step>

  <Step>

### 选择快照

从您要恢复的时间段中选择快照

</Step>

  <Step>

### 浏览并选择

浏览文件树，选择要恢复的文件/文件夹

</Step>

  <Step>

### 选择恢复位置

决定是恢复到原始位置还是指定新路径

</Step>

  <Step>

### 恢复

点击**全部恢复**恢复所有内容，或先选择特定文件再仅恢复所选内容

</Step>
</Steps>

<Callout type="info">
  有关完整的恢复工作流程、覆盖模式、下载与恢复以及挂载卷权限注意事项，请参阅[恢复数据](/docs/guides/restoring)。
</Callout>

### 暂停备份任务

要临时停止备份运行：

1. 导航到备份任务
2. 关闭**启用**开关
3. 任务将跳过计划的运行，直到重新启用

### 查看备份详情

要检查备份运行情况：

1. 点击备份任务
2. 查看当前的**状态**、**上次备份**和**快照**部分
3. 点击快照以查看其文件浏览器和恢复操作
4. 如需更深入的故障排查，请检查 `docker compose logs -f zerobyte`

### 删除旧快照

备份会根据保留策略自动清理，但您也可以手动删除快照：

1. 导航到备份任务
2. 选择要删除的快照
3. 点击**删除快照**
4. 确认删除

<Callout type="warn">
  删除快照是不可逆的操作。请确保在删除前您不再需要这些数据。
</Callout>

## 故障排除

### 备份失败：权限被拒绝

请确保卷目录具有正确的权限：

```bash
# 在宿主机系统上
sudo chmod -R 755 /path/to/your/data
```

### 备份运行缓慢

请检查：

- 网络带宽（对于云仓库）
- 磁盘 I/O 性能
- 如果上传大量可压缩文件，请启用压缩
- 检查是否启用了仓库上传/下载速度限制

### 仓库初始化失败

请确认：

- 仓库路径可写入
- 对于云仓库，凭据是否正确
- 与云存储的网络连接是否正常

### 找不到已挂载的目录

请确保：

- 目录已在 `docker-compose.yml` 中挂载
- 挂载后已重启容器：`docker compose restart`
- 路径使用的是容器路径，而非宿主机路径
