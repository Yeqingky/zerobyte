Zerobyte 以 Docker 容器方式运行，需要在服务器上安装 Docker 和 Docker Compose。

## 前提条件

<Steps>
<Step>

### 安装 Docker

确保服务器上已安装 Docker。请访问 [docs.docker.com](https://docs.docker.com/get-docker/) 获取适用于你平台的安装说明。

</Step>

<Step>

### 安装 Docker Compose

Docker Compose 是编排所必需的。它已包含在 Docker Desktop 中，也可以在 Linux 服务器上单独安装。

</Step>

<Step>

### 检查安装

通过运行以下命令验证安装：

```bash
docker --version
docker compose version
```

</Step>
</Steps>

## 基本安装

标准安装包含远程挂载支持（NFS、SMB、WebDAV、SFTP），需要容器拥有提升的权限。

### 1. 创建 docker-compose.yml

使用以下配置创建 `docker-compose.yml` 文件：

```yaml docker-compose.yml
services:
  zerobyte:
    image: ghcr.io/nicotsx/zerobyte:v0.36
    container_name: zerobyte
    restart: unless-stopped
    cap_add:
      - SYS_ADMIN
    ports:
      - "4096:4096"
    devices:
      - /dev/fuse:/dev/fuse
    environment:
      - TZ=Europe/Zurich # 在此处设置你的时区
      - BASE_URL=http://localhost:4096 # 访问 Zerobyte 的 URL
      - APP_SECRET=94bad46...c66e25d5c2b # 使用 `openssl rand -hex 32` 生成你自己的密钥
    volumes:
      - /etc/localtime:/etc/localtime:ro
      - /var/lib/zerobyte:/var/lib/zerobyte
```

<Callout type="warn">
**安全说明**：`SYS_ADMIN` 权限和 `/dev/fuse` 设备是挂载远程文件系统（NFS、SMB、WebDAV、SFTP）所必需的。如果只需要本地目录备份，请参见下面的[简化安装](#simplified-installation-no-remote-mounts)部分。

远程挂载虽然方便，但可能会暴露经过转换的所有权、权限和 ACL 元数据，而非源系统的原始视图。在对元数据敏感的备份中使用挂载卷之前，请阅读[挂载共享与权限](/docs/guides/mounted-shares-and-acls)。
</Callout>

### 2. 配置环境变量

更新 `docker-compose.yml` 中的环境变量：

**生成 APP_SECRET：**

```bash
openssl rand -hex 32
```

**配置示例：**

```yaml
environment:
  - TZ=America/New_York
  - BASE_URL=http://192.168.1.100:4096
  - APP_SECRET=a1b2c3d4e5f6...  # openssl 命令的输出
```

#### 必需的环境变量

| 变量 | 描述 | 示例 |
|----------|-------------|---------|
| `BASE_URL` | **必需。**访问 Zerobyte 的基础 URL。用于 Cookie 安全和 CORS。 | `http://localhost:4096` 或 `https://zerobyte.example.com` |
| `APP_SECRET` | **必需。**用于加密数据库中敏感数据的 32+ 位随机密钥。使用 `openssl rand -hex 32` 生成。 | `94bad46...c66e25d5c2b` |
| `TZ` | **推荐。**用于准确备份调度的时区。 | `Europe/Zurich`、`America/New_York`、`UTC` |

<Callout type="info">
`BASE_URL` 决定 Cookie 安全行为：

* **HTTP 或 IP 地址**：安全 Cookie 已禁用（允许本地访问）
* **HTTPS 加域名**：安全 Cookie 已启用（生产环境必需）
</Callout>

#### 可选的环境变量

| 变量 | 描述 | 默认值 |
|----------|-------------|---------|
| `PORT` | 容器内 Web 界面监听的端口 | `4096` |
| `RESTIC_HOSTNAME` | Restic 在快照中使用的主机名 | `zerobyte` |
| `TRUSTED_ORIGINS` | 额外受信任 CORS 来源的逗号分隔列表 | (无) |
| `WEBHOOK_TIMEOUT` | 备份 Webhook 请求的超时时间（秒） | `60` |
| `LOG_LEVEL` | 日志详细级别：`debug`、`info`、`warn`、`error` | `info` |
| `SERVER_IDLE_TIMEOUT` | 服务器空闲超时时间（秒） | `60` |
| `RCLONE_CONFIG_DIR` | 容器内 rclone 配置目录的路径 | `/root/.config/rclone` |

### 3. 配置卷挂载

必需卷挂载用于存储 Zerobyte 的数据：

```yaml
volumes:
  - /etc/localtime:/etc/localtime:ro
  - /var/lib/zerobyte:/var/lib/zerobyte
```

<Callout type="warn">
**重要**：请勿将 `/var/lib/zerobyte` 指向网络共享。这会导致权限问题和严重的性能下降。请始终使用本地存储。
</Callout>

<Callout type="info">
**TrueNAS 用户**：`/var/lib` 路径在 TrueNAS 上是临时的，系统升级时会重置。请改为创建专用的 ZFS 数据集：

```yaml
volumes:
  - /etc/localtime:/etc/localtime:ro
  - /mnt/tank/docker/zerobyte:/var/lib/zerobyte
```

这样可以确保你的配置、加密密钥和数据库在升级后持续存在。
</Callout>

### 4. 启动 Zerobyte

使用 Docker Compose 启动容器：

```bash
docker compose up -d
```

验证容器是否正在运行：

```bash
docker compose ps
docker compose logs -f zerobyte
```

### 5. 访问 Web 界面

容器运行后，通过 `BASE_URL` 中指定的 URL 访问 Zerobyte：

```
http://<your-server-ip>:4096
```

<Callout type="info">
首次访问时，系统会提示你创建管理员账户。该账户将拥有所有备份管理功能的完全访问权限。
</Callout>

## 简化安装（无远程挂载）

如果只需要备份本地挂载的目录，不需要远程共享挂载（NFS、SMB、WebDAV、SFTP），可以使用低权限部署方案：

```yaml docker-compose.yml
services:
  zerobyte:
    image: ghcr.io/nicotsx/zerobyte:v0.36
    container_name: zerobyte
    restart: unless-stopped
    ports:
      - "4096:4096"
    environment:
      - TZ=Europe/Zurich
      - BASE_URL=http://localhost:4096
      - APP_SECRET=94bad46...c66e25d5c2b
    volumes:
      - /etc/localtime:/etc/localtime:ro
      - /var/lib/zerobyte:/var/lib/zerobyte
      - /path/to/your/directory:/mydata
```

**权衡：**

<ul className="my-4 list-none space-y-2 pl-0">
  <li className="flex items-start gap-2">
    <Check className="mt-1.5 h-4 w-4 shrink-0 text-green-600" />
    <span>通过移除 `SYS_ADMIN` 权限提高安全性</span>
  </li>
  <li className="flex items-start gap-2">
    <Check className="mt-1.5 h-4 w-4 shrink-0 text-green-600" />
    <span>支持挂载到容器中的本地目录</span>
  </li>
  <li className="flex items-start gap-2">
    <Check className="mt-1.5 h-4 w-4 shrink-0 text-green-600" />
    <span>仍支持所有仓库类型（local、S3、GCS、Azure、rclone）</span>
  </li>
  <li className="flex items-start gap-2">
    <X className="mt-1.5 h-4 w-4 shrink-0 text-red-600" />
    <span>无法从 Zerobyte 内部挂载远程共享（NFS、SMB、WebDAV、SFTP）</span>
  </li>
</ul>

<Callout type="info">
如果后续需要远程挂载功能，可以更新 `docker-compose.yml` 重新添加 `cap_add: SYS_ADMIN` 和 `devices: /dev/fuse:/dev/fuse` 指令。

如果你的目标是尽可能接近真正的复制，请尽可能优先使用本地绑定挂载目录。有关实际权衡，请参阅[挂载共享与权限](/docs/guides/mounted-shares-and-acls)。
</Callout>

## 挂载本地目录

要备份主机系统的目录，请将其挂载到容器中：

```yaml
volumes:
  - /etc/localtime:/etc/localtime:ro
  - /var/lib/zerobyte:/var/lib/zerobyte
  - /path/to/your/photos:/photos
  - /path/to/your/documents:/documents
  - /path/to/your/media:/media
```

<Callout type="info">
如果希望防止 Zerobyte 修改源数据，请使用只读挂载（`:ro`）。但使用只读挂载后，你将无法将文件恢复到原始位置。
</Callout>

添加卷挂载后，重启容器：

```bash
docker compose down
docker compose up -d
```

挂载的目录将在容器内的指定路径（如 `/photos`、`/documents`、`/media`）可用。

## 高级配置

### 使用 Docker Secrets 管理敏感数据

如果使用自动配置，Zerobyte 可以在将解析后的值加密存储到数据库之前，从环境变量或 Docker Secret 文件中解析密钥：

```yaml docker-compose.yml
services:
  zerobyte:
    image: ghcr.io/nicotsx/zerobyte:v0.36
    container_name: zerobyte
    restart: unless-stopped
    cap_add:
      - SYS_ADMIN
    devices:
      - /dev/fuse:/dev/fuse
    ports:
      - "4096:4096"
    environment:
      - TZ=Europe/Zurich
      - BASE_URL=http://localhost:4096
      - APP_SECRET=94bad46...c66e25d5c2b
      - S3_ACCESS_KEY=your-access-key
      - S3_SECRET_KEY=your-secret-key
    secrets:
      - smb_password
    volumes:
      - /etc/localtime:/etc/localtime:ro
      - /var/lib/zerobyte:/var/lib/zerobyte

secrets:
  smb_password:
    file: ./secrets/smb_password.txt
```

编写自动配置文件时，可以引用这些密钥：

* `env://S3_SECRET_KEY` - 从环境变量 `S3_SECRET_KEY` 解析
* `file://smb_password` - 从 `/run/secrets/smb_password` 解析

<Callout type="info">
  这些引用目前仅在自动配置期间解析。UI 中的常规卷和仓库表单需要实际的密钥值。有关完整工作流程，请参阅[自动配置指南](/docs/guides/provisioning)。
</Callout>

### 挂载 rclone 配置

要使用基于 rclone 的仓库（Google Drive、Dropbox、OneDrive 等），请挂载你的 rclone 配置：

<Steps>
<Step>

### 在主机上配置 rclone

在主机系统上安装并配置 rclone：

```bash
curl https://rclone.org/install.sh | sudo bash
rclone config
```

</Step>

<Step>

### 将配置挂载到容器中

更新你的 `docker-compose.yml`：

```yaml
volumes:
  - /etc/localtime:/etc/localtime:ro
  - /var/lib/zerobyte:/var/lib/zerobyte
  - ~/.config/rclone:/root/.config/rclone:ro
```

</Step>

<Step>

### 重启容器

```bash
docker compose down && docker compose up -d
```

</Step>
</Steps>

现在，在 Zerobyte 中创建仓库时，你的 rclone 远程端点将可用。
