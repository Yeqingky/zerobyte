---
title: Rclone 集成
description: 将 rclone 与 Zerobyte 结合使用，连接 40+ 云存储提供商作为仓库和存储卷
---

Rclone 是一个命令行工具，可连接到超过 40 家云存储提供商，包括 Google Drive、Dropbox、OneDrive、Box、pCloud、Mega 等。Zerobyte 以两种方式利用 rclone：

- **仓库**，将加密备份存储在任何 rclone 支持的云提供商上
- **Volume**，将云存储挂载为数据源，以便进行备份

本指南涵盖安装 rclone、将其连接到 Zerobyte，以及将其用于仓库和 Volume。

## 设置 rclone

在 Zerobyte 可以使用 rclone 之前，您需要先在主机上安装并配置它。

<Steps>
<Step>

### 安装 rclone

在主机系统上安装 rclone：

```bash
curl https://rclone.org/install.sh | sudo bash
```

或者，访问 [rclone.org/install](https://rclone.org/install/) 获取特定平台的软件包。

</Step>

<Step>

### 配置远程

运行交互式配置向导：

```bash
rclone config
```

按照提示添加新的远程。每个提供商都有自己的设置流程，rclone 会根据需要引导您完成身份验证、API 密钥或 OAuth。

</Step>

<Step>

### 验证远程

列出所有已配置的远程：

```bash
rclone listremotes
```

</Step>

<Step>

### 测试连接

列出远程上的顶级目录以确认一切正常：

```bash
rclone lsd myremote:
```

将 `myremote` 替换为您在配置期间选择的名称。

</Step>
</Steps>

## 将 rclone 配置挂载到容器中

Zerobyte 需要访问您的 rclone 配置文件才能发现和使用您的远程。将其作为只读卷挂载到您的 `docker-compose.yml` 中：

```yaml docker-compose.yml
volumes:
  - /etc/localtime:/etc/localtime:ro
  - /var/lib/zerobyte:/var/lib/zerobyte
  - ~/.config/rclone:/root/.config/rclone:ro
```

然后重启容器以应用更改：

```bash
docker compose down && docker compose up -d
```

<Callout type="info">
对于非 root 容器环境（例如 TrueNAS），配置必须挂载到正确的用户主目录。相应地设置 `RCLONE_CONFIG_DIR` 环境变量：

```yaml docker-compose.yml
environment:
  - RCLONE_CONFIG_DIR=/home/appuser/.config/rclone
volumes:
  - ~/.config/rclone:/home/appuser/.config/rclone:ro
```
</Callout>

## 将 rclone 用于仓库

Rclone 仓库让您可以将加密备份存储在任何 rclone 支持的提供商上。

<Steps>
<Step>

### 打开创建仓库表单

导航到侧边栏中的**仓库**，点击**创建仓库**。

</Step>

<Step>

### 选择 rclone 作为类型

从仓库类型下拉菜单中选择 **rclone**。

</Step>

<Step>

### 选择远程

从下拉列表中选择您的远程。Zerobyte 从已挂载的 rclone 配置中读取所有已配置的远程。

</Step>

<Step>

### 指定路径

输入远程内用于存储备份的路径（例如 `backups/zerobyte`）。

</Step>

<Step>

### 创建仓库

点击**创建并初始化**。Zerobyte 将在远程上初始化一个新的加密 Restic 仓库。

</Step>
</Steps>

<Callout type="warn">
消费者级云服务（如 Google Drive 和 Dropbox）有 API 速率限制，不适合高 I/O 工作负载。它们适用于个人或轻度备份使用，但对于生产环境或关键数据，请使用适当的对象存储，如 S3、Backblaze B2 或 Cloudflare R2。
</Callout>

## 将 rclone 用于 Volume

Rclone Volume 将云存储挂载为容器内的文件系统，允许您备份存储在云中的数据。这需要提升的容器权限。

<Steps>
<Step>

### 确保 FUSE 支持

您的 `docker-compose.yml` 必须包含 `SYS_ADMIN` 和 `/dev/fuse`：

```yaml docker-compose.yml
cap_add:
  - SYS_ADMIN
devices:
  - /dev/fuse:/dev/fuse
```

</Step>

<Step>

### 创建新 Volume

导航到侧边栏中的**Volume**，点击**创建 Volume**。

</Step>

<Step>

### 选择 rclone 作为类型

从 Volume 类型下拉菜单中选择 **rclone**。

</Step>

<Step>

### 选择远程和路径

从下拉列表中选择您的远程，并指定要挂载的远程内路径（例如，`/` 表示整个远程，或 `/photos` 表示子目录）。

</Step>

<Step>

### 保存并挂载

点击**创建**。Zerobyte 将挂载远程作为 FUSE 文件系统，使其可用于备份任务。

</Step>
</Steps>

<Callout type="warn">
Rclone Volume **仅适用于 Linux**。macOS 和 Windows 上的 Docker 不支持容器内的 FUSE 挂载。如果您在 macOS 或 Windows 上运行 Docker Desktop，rclone Volume 将无法工作。
</Callout>

## 使用 SSH 密钥的 SFTP 远程

如果您的 rclone 远程使用带 `key_file` 身份验证的 SFTP，则 rclone 配置中的路径指向主机文件系统上的某个位置，而该位置在容器内不可访问。有三种解决方法。

<Tabs items={["挂载 SSH 密钥", "将密钥嵌入配置", "SSH 代理"]}>
<Tab value="Mount SSH keys">

**推荐。** 将您的 SSH 密钥挂载到容器中，以便 rclone 可以在预期路径找到它们：

```yaml docker-compose.yml
volumes:
  - ~/.ssh:/root/.ssh:ro
```

确保 rclone 配置中的 `key_file` 路径与容器路径匹配（例如 `/root/.ssh/id_rsa`）。

</Tab>

<Tab value="Embed key in config">

不使用 `key_file` 引用文件，而是使用 `key_pem` 字段将私钥直接嵌入到 rclone 配置中。在主机上运行 `rclone config` 并选择内联粘贴密钥的选项。

这样可以避免任何文件路径问题，因为密钥直接存储在 rclone 配置文件中。

</Tab>

<Tab value="SSH agent">

将主机的 SSH 代理套接字转发到容器中：

```yaml docker-compose.yml
environment:
  - SSH_AUTH_SOCK=/ssh-agent
volumes:
  - ${SSH_AUTH_SOCK}:/ssh-agent
```

这允许 rclone 使用加载到主机 SSH 代理中的密钥进行身份验证，而无需暴露私钥文件。

</Tab>
</Tabs>

## 刷新 OAuth 令牌

使用 OAuth 的 rclone 远程（Google Drive、Dropbox、OneDrive、Box 等）将身份验证令牌存储在 rclone 配置文件中。这些令牌会定期过期，需要刷新。

<Steps>
<Step>

### 在主机上重新认证

在主机上运行 rclone config 命令，并按照提示重新授权远程：

```bash
rclone config reconnect myremote:
```

</Step>

<Step>

### 重启容器

刷新令牌后，重启 Zerobyte 容器以加载更新后的配置：

```bash
docker compose restart
```

</Step>
</Steps>

<Callout type="info">
如果以只读卷（`:ro`）挂载 rclone 配置，主机上更新后的令牌在重启后立即可用于容器。无需手动复制文件。
</Callout>

## 故障排除

### 没有可用的远程

如果在创建仓库或 Volume 时 rclone 远程下拉列表为空：

- 验证 rclone 配置是否正确挂载。检查主机上是否存在 `~/.config/rclone/rclone.conf`。
- 确认 `docker-compose.yml` 中的挂载路径与容器内的预期位置匹配（默认为 `/root/.config/rclone`，或 `RCLONE_CONFIG_DIR` 中设置的路径）。
- 更新卷挂载后重启容器。

### 创建文件系统失败

这通常意味着 rclone 无法与远程进行身份验证：

- 对于 OAuth 远程（Google Drive、Dropbox、OneDrive），令牌可能已过期。在主机上使用 `rclone config reconnect myremote:` 重新认证，然后重启容器。
- 对于基于密钥的远程（S3、SFTP），验证 rclone 配置中的凭据是否正确。
- 直接在主机上测试远程：`rclone lsd myremote:` 以隔离问题是 rclone 本身还是 Zerobyte 引起的。

### rclone Volume 的 FUSE 挂载错误

如果在创建 rclone Volume 时看到与 FUSE 相关的错误：

- 确保您的 `docker-compose.yml` 包含 `cap_add: [SYS_ADMIN]` 和 `devices: [/dev/fuse:/dev/fuse]`。
- 确认您在 Linux 主机上运行。FUSE 挂载在 macOS 或 Windows 上的 Docker 中无法工作。
- 检查主机上是否存在 `/dev/fuse`：`ls -la /dev/fuse`。

<Callout type="info">
有关更详细的故障排除步骤，请参阅[故障排除](/docs/troubleshooting)页面。
</Callout>
