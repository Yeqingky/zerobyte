通过 Tailscale 边车容器运行 Zerobyte，使 Web UI 可通过您的私有 tailnet 访问，无需向公共互联网暴露端口。

## 什么是 Tailscale？

[Tailscale](https://tailscale.com/) 是一个基于 WireGuard 的 mesh VPN。它将设备和容器连接到一个私有网络（"tailnet"），无需在路由器上打开入站端口。

在此设置中，Tailscale 充当 Zerobyte 前面的安全访问层：

- 您通过节点的 tailnet IP 或 MagicDNS 名称访问 Zerobyte
- 可以使用 Tailscale ACL 和标签限制访问
- 无需端口转发或公网暴露

## 前提条件

- Docker 和 Docker Compose
- 一个 [Tailscale 账户](https://tailscale.com/) 和一个[认证密钥](https://login.tailscale.com/admin/settings/keys)

## 设置步骤

此示例使用边车网络模式，Zerobyte 共享 Tailscale 容器的网络命名空间。

<Steps>
<Step>

### 创建 docker-compose.yml

```yaml docker-compose.yml
services:
  tailscale:
    image: tailscale/tailscale:stable
    container_name: zerobyte-tailscale
    hostname: ${TS_HOSTNAME:-zerobyte}
    restart: unless-stopped
    cap_add:
      - NET_ADMIN
    devices:
      - /dev/net/tun:/dev/net/tun
    environment:
      - TS_AUTHKEY=${TS_AUTHKEY}
      - TS_STATE_DIR=/var/lib/tailscale
      - TS_USERSPACE=${TS_USERSPACE:-false}
      - TS_EXTRA_ARGS=${TS_EXTRA_ARGS:-}
    volumes:
      - /var/lib/tailscale:/var/lib/tailscale
    # 移除本节以使 Zerobyte 仅可通过 Tailscale 访问
    ports:
      - "4096:4096"

  zerobyte:
    image: ghcr.io/nicotsx/zerobyte:latest
    container_name: zerobyte
    restart: unless-stopped
    # 如果需要远程挂载（NFS/SMB/WebDAV），取消注释：
    # cap_add:
    #   - SYS_ADMIN
    # devices:
    #   - /dev/fuse:/dev/fuse
    network_mode: service:tailscale
    depends_on:
      - tailscale
    environment:
      - TZ=${TZ:-UTC}
      - BASE_URL=http://${TS_HOSTNAME:-zerobyte}:4096
      - APP_SECRET=${APP_SECRET}
    volumes:
      - /etc/localtime:/etc/localtime:ro
      - /var/lib/zerobyte:/var/lib/zerobyte
```

</Step>
<Step>

### 创建 .env 文件

```bash .env
TS_AUTHKEY=tskey-auth-xxxxx
TS_HOSTNAME=zerobyte
APP_SECRET=your-secret-here  # 使用以下命令生成：openssl rand -hex 32
TZ=UTC
# 可选：
# TS_EXTRA_ARGS=--advertise-tags=tag:backup
# TS_USERSPACE=false
```

</Step>
<Step>

### 启动服务栈

```bash
docker compose up -d
```

</Step>
<Step>

### 在 Tailscale 中验证

确认节点出现在您的 [Tailscale 管理控制台](https://login.tailscale.com/admin/machines)中。如果您的策略需要批准，请批准该节点。

</Step>
</Steps>

## 访问 Zerobyte

运行后，通过您的 tailnet 访问 Zerobyte：

- **通过 IP**：`http://<tailscale-ip>:4096`
- **通过 MagicDNS**：`http://zerobyte:4096`（如果启用了 MagicDNS）

要使 Zerobyte **仅**通过 Tailscale 可访问，请移除 `tailscale` 服务中的 `ports:` 部分。

## 内核模式与用户空间模式

| 模式 | 要求 | 适用场景 |
|------|------|----------|
| **内核**（默认） | `/dev/net/tun`、`NET_ADMIN` | Linux 服务器，最佳性能 |
| **用户空间** | 无需特殊设备 | Docker Desktop，受限主机 |

使用用户空间模式：
1. 在 `.env` 中设置 `TS_USERSPACE=true`
2. 移除 tailscale 服务中的 `devices: /dev/net/tun` 部分

## 仅限 Tailscale 访问

如果您希望 Zerobyte **仅**通过 Tailscale 可访问（而非本地网络），请移除 `tailscale` 服务中的 `ports:` 部分。Zerobyte 仍然可以访问互联网和您的 LAN 以进行备份操作，但 UI 只能通过 Tailscale 访问。

## 故障排除

- **TUN 设备错误**：确保主机上存在 `/dev/net/tun`。如果不存在，请切换到用户空间模式。
- **节点未显示**：检查认证密钥是否有效且未过期。
- **ACL 问题**：设置 `TS_EXTRA_ARGS=--advertise-tags=tag:backup` 并在 Tailscale 管理控制台中配置 ACL。

验证 tailnet 地址：

```bash
docker exec zerobyte-tailscale tailscale status
docker exec zerobyte-tailscale tailscale ip -4
```
