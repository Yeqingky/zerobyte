Zerobyte 通过环境变量和 Docker Compose 设置进行配置。本页涵盖所有可用选项。

## 环境变量

### 必需

| 变量 | 描述 | 示例 |
|----------|-------------|---------|
| `BASE_URL` | 访问 Zerobyte 的 URL。控制 Cookie 安全和 CORS 行为。 | `http://localhost:4096` 或 `https://zerobyte.example.com` |
| `APP_SECRET` | 用于加密数据库中敏感数据的随机密钥（32+ 字符）。使用 `openssl rand -hex 32` 生成。设置此项或 `APP_SECRET_FILE`，但不能同时设置两者。 | `94bad46e...c66e25d5c2b` |
| `APP_SECRET_FILE` | `APP_SECRET` 的替代项。包含应用密钥的文件的路径，适用于 Docker 或 Kubernetes Secrets。设置此项或 `APP_SECRET`，但不能同时设置两者。 | `/run/secrets/app_secret` |

<Callout type="warn">
  切勿分享或提交你的 `APP_SECRET`。如果丢失，加密数据（为卷和仓库存储的凭据）将无法恢复。
</Callout>

<Callout type="info">
  Zerobyte 会读取 `APP_SECRET_FILE` 的内容，去除周围空白和换行，并应用与 `APP_SECRET` 相同的 32-256 字符要求。
</Callout>

### 推荐

| 变量 | 描述 | 默认值 |
|----------|-------------|---------|
| `TZ` | 容器的时区。**对于准确的备份调度很重要。** | `UTC` |

### 可选

| 变量 | 描述 | 默认值 |
|----------|-------------|---------|
| `PORT` | 容器内 Web 界面和 API 监听的端口。 | `4096` |
| `RESTIC_HOSTNAME` | Restic 创建快照时使用的主机名。如果在 Docker 中设置了自定义主机名，则会自动检测。 | `zerobyte` |
| `TRUST_PROXY` | 设置为 `true` 以信任来自反向代理的 `X-Forwarded-For` 头。 | `false` |
| `TRUSTED_ORIGINS` | 额外受信任 CORS 来源的逗号分隔列表。 | (无) |
| `WEBHOOK_ALLOWED_ORIGINS` | 允许用于备份 Webhook 和出站 HTTP 通知目标地址的 HTTP 来源的逗号分隔列表。 | (无) |
| `WEBHOOK_TIMEOUT` | 备份 Webhook 请求的超时时间（秒）。 | `60` |
| `LOG_LEVEL` | 日志详细级别：`debug`、`info`、`warn`、`error`。 | `info` |
| `SERVER_IDLE_TIMEOUT` | 服务器空闲超时时间（秒）。 | `60` |
| `RCLONE_CONFIG_DIR` | 容器内 rclone 配置目录的路径。 | `/root/.config/rclone` |
| `PROVISIONING_PATH` | 包含运维管理的仓库和卷的 JSON 文件路径，用于在启动时同步。 | (无) |

## 使用 APP_SECRET_FILE

如果不希望直接将应用密钥放在 `environment:` 中，可以将其作为文件挂载，并将 `APP_SECRET_FILE` 指向该路径。

```yaml docker-compose.yml
services:
  zerobyte:
    environment:
      - BASE_URL=https://zerobyte.example.com
      - APP_SECRET_FILE=/run/secrets/app_secret
    secrets:
      - app_secret

secrets:
  app_secret:
    file: ./secrets/app_secret.txt
```

使用以下命令生成密钥文件：

```bash
mkdir -p ./secrets
openssl rand -hex 32 > ./secrets/app_secret.txt
```

不要同时设置 `APP_SECRET`。如果两者都配置，Zerobyte 将无法启动。

## Docker Compose 设置

### 卷挂载

Zerobyte 必需的卷挂载：

```yaml
volumes:
  # 将容器时间与主机同步（推荐）
  - /etc/localtime:/etc/localtime:ro

  # Zerobyte 数据目录（数据库、加密密钥、本地仓库）
  - /var/lib/zerobyte:/var/lib/zerobyte
```

<Callout type="warn">
  **请勿**将 `/var/lib/zerobyte` 指向网络共享。这会导致权限问题和严重的性能下降。请始终使用本地存储。
</Callout>

<Callout type="info">
  **TrueNAS 用户**：`/var/lib` 路径是临时的，在系统升级时会重置。请改为创建专用的 ZFS 数据集：

  ```yaml
  volumes:
    - /mnt/tank/docker/zerobyte:/var/lib/zerobyte
  ```
</Callout>

#### 额外的卷挂载

| 挂载 | 用途 |
|-------|---------|
| `/path/to/data:/data:ro` | 挂载主机目录以进行本地目录备份（使用 `:ro` 表示只读） |
| `~/.config/rclone:/root/.config/rclone:ro` | 挂载 rclone 配置以用于基于 rclone 的仓库和卷 |
| `~/.ssh:/root/.ssh:ro` | 挂载用于使用 `key_file` 的 rclone SFTP 远程端点的 SSH 密钥 |
| `./provisioning.json:/config/provisioning.json:ro` | 挂载用于运维管理资源的自动配置文件 |

### 容器权限

Zerobyte 根据需求支持两种部署模式：

#### 完整安装（带远程挂载）

直接在 Zerobyte 中挂载 NFS、SMB、WebDAV 和 SFTP 卷所需：

```yaml
cap_add:
  - SYS_ADMIN
devices:
  - /dev/fuse:/dev/fuse
```

#### 简化安装（仅本地目录）

如果只需要本地目录备份，则无需特殊权限：

```yaml
# 不需要 cap_add 或 devices
ports:
  - "4096:4096"
```

### 端口配置

默认情况下，Zerobyte 监听 4096 端口：

```yaml
ports:
  - "4096:4096"
```

仅绑定到 localhost（使用反向代理时推荐）：

```yaml
ports:
  - "127.0.0.1:4096:4096"
```

## Cookie 安全

`BASE_URL` 决定身份验证 Cookie 的行为方式：

| BASE_URL | Cookie 行为 |
|----------|----------------|
| `http://192.168.1.50:4096` | 安全 Cookie **已禁用**，允许通过 HTTP 登录 |
| `http://localhost:4096` | 安全 Cookie **已禁用**，允许本地开发 |
| `https://zerobyte.example.com` | 安全 Cookie **已启用**，需要 HTTPS |

<Callout type="info">
  如果 `BASE_URL` 以 `https://` 开头，浏览器将仅通过 HTTPS 连接发送身份验证 Cookie。纯 HTTP 访问可能显示登录页面，但身份验证将失败。
</Callout>

<Callout type="warn">
  `TRUSTED_ORIGINS` 仅允许 CORS 的额外来源。当 `BASE_URL` 为 HTTPS 时，它**不会**禁用安全 Cookie 或使 HTTP 访问生效。
</Callout>

## 密钥引用

自动配置卷或仓库时，敏感字段支持密钥引用：

| 引用 | 解析来源 | 示例 |
|-----------|--------------|---------|
| `env://VARIABLE_NAME` | 容器环境变量 | `env://S3_SECRET_KEY` |
| `file://secret_name` | 位于 `/run/secrets/secret_name` 的 Docker Secret | `file://smb_password` |

这使你可以将凭据保留在部署配置中，而不是直接写入自动配置文件中。

<Callout type="info">
  UI 中的标准卷和仓库表单当前会存储你输入的值（静态加密）。`env://` 和 `file://` 引用仅在自动配置期间解析。
</Callout>

### Docker Secrets 示例

```yaml docker-compose.yml
services:
  zerobyte:
    environment:
      - S3_ACCESS_KEY=AKIAIOSFODNN7EXAMPLE
    secrets:
      - s3_secret_key
    volumes:
      - /var/lib/zerobyte:/var/lib/zerobyte

secrets:
  s3_secret_key:
    file: ./secrets/s3_secret_key.txt
```

在自动配置文件中，按如下方式引用：
- 访问密钥：`env://S3_ACCESS_KEY`
- 密钥：`file://s3_secret_key`

## 更新 Zerobyte

更新到新版本：

```bash
docker compose pull
docker compose up -d
docker compose logs -f zerobyte
```

<Callout type="warn">
  更新前请务必查看[发布说明](https://github.com/nicotsx/zerobyte/releases)，特别是 v0.x.x 版本可能包含破坏性变更。
</Callout>
